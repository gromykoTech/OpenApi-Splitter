# Установка и запуск OpenAPI Splitter

## Требования

### Минимальные требования

- **Node.js** >= 18
- **Docker** и **Docker Compose** (для основного способа запуска)

### Рекомендуемая версия

- **Node.js 20** (используется в Docker-образе)

## Запуск через Docker (рекомендуется)

Основной и рекомендуемый способ запуска приложения.

### Команды

```bash
docker compose build
docker compose up
```

### Доступ к приложению

После запуска приложение будет доступно по адресу:

```
http://localhost:5173
```

### Что происходит при запуске

1. **Сборка образа**: создается Docker-образ на основе `node:20-alpine`
2. **Установка зависимостей**: выполняется `yarn install --frozen-lockfile`
3. **Запуск контейнера**: запускается dev-сервер Vite с hot-reload
4. **Монтирование volumes**: исходный код монтируется в контейнер для разработки

### Остановка

Для остановки контейнера используйте:

```bash
docker compose down
```

## Локальный запуск (опционально)

Локальный запуск возможен, но рекомендуется использовать Docker для консистентности окружения.

### Установка зависимостей

```bash
yarn install
```

### Запуск dev-сервера

```bash
yarn dev
```

Приложение будет доступно по адресу: `http://localhost:5173`

### Сборка для production

```bash
yarn build
```

Собранные файлы будут находиться в папке `dist/`.

### Просмотр production-сборки

```bash
yarn preview
```

### Линтинг

```bash
yarn lint
```

## Примечания

- Для разработки рекомендуется использовать Docker, так как это обеспечивает консистентное окружение
- При локальном запуске убедитесь, что версия Node.js соответствует требованиям
- Порт 5173 может быть изменен в `docker-compose.yml` при необходимости
